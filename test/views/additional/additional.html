Loader

If you want to add loader in your application then you can add your loader html to your index.html and configure router to show and hide when page is fully loaded.

<!doctype html>
<html lang="en">

<head>
    <title>Your Page</title>
    <meta content="http://localhost:3000/" name="base_url" />
</head>

<body>
    <nav class="....">
        ...
    </nav>

    <div class="container">
        
        <div class="progress d-none">
            <div class="progress-bar progress-bar-striped progress-bar-animated" role="progressbar" aria-valuenow="100" aria-valuemin="0"
                 aria-valuemax="100" style="width: 100%"></div>
        </div>
        
        <div id="app"></div>
    </div>

    <footer>
        ...
    </footer>

    <script src="js/sp-router/sp-router.js"></script>
    <script src="js/routerConfig.js"></script>
</body>

</html>

Then configure in routerConfig.js

var router = new SPRouter({
    content: "#app",
    defaultRoute: "/"
    routes: [
        {
            title: "Products",
            route: "/products",
            template: "views/products/products.html",
            script: "views/products/products.js"
        },
        {
            title: "Product",
            route: "/products/:productId/view",
            template: "views/products/product.html",
            script: "views/products/product.js"
        }
    ],
    loader: {
        selector: ".progress", // your loader class or id selector
    },
});

This configuration will give show/hide behavior with display:none/block; inline style.

If you to show/hide with custom class then add showHideClass key to loader configuration

    loader: {
        selector: ".progress",
        showHideClass: "d-none" // your class to show and hide
    },

By default your loader must be hidden (to have your class for hidding) and when router will start loading then will remove your class and when will finish with loading will add your class to hide page loader.

By default when loader is showed up div#app content is hiding. To prevent this behavior do not came up provide loadContentHide into loader configuration.

    loader: {
        selector: ".progress",
        showHideClass: "d-none",
        loadContentHide: false
    },

Lazy loading

If you want to add some delay when router is loading add lazy array objects in routerConfig.js configuration

    lazy: {
        loading: true
    },

By default lazy loading set to true has additional 500 milliseconds.

If you want to set custom delay interval set duration object into lazy array.

    lazy: {
        loading: true,
        duration: 1000 // in milliseconds
    },
